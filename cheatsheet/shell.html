      What are the most basic Linux commands every developer should know?

    1. man (manual page, e.g., man rm)

    2. cd (change directory)

    3. ls (list contents of directory, see man ls for more options, e.g., ls -1, just print file names)

    4. mv (move/rename file or directory)

    5. cp (to copy files)

    6. clear (clears terminal window)

    7. rm (remove file or directory)

    8. echo (print something to stdout)

    9. cat (print an entire file to stdout)

    10. tail (see man tail for options, see the last 10 lines of a file by default), which is also useful
      for leaving a file-handle open and seeing as the file is modified, e.g., tail -f some_file

    11. less (inspect a file page by page or line by line)

    12. grep (see man grep, finds files that contain a string, also see man egrep for finding files that contain a regular expression)

    13. sort (see man sort for sorting options)

    14. uniq (get a unique listing of elements, e.g., cat some_file | awk -F',' '{print $2}' | sort | uniq)

    15. history (see recent bash history, also see ~/.bash_history)

    16. view (read-only vim)

    17. ln (to create links to files)

    18. chmod (change permissions, see Linux and Unix chmod command help and examples)

    19. chmod (change permissions, see Linux and Unix chmod command help and examples)

    20. chown (change user and group ownership)

    21. chgrp (change just group ownership)

    22. top (for monitoring processes)

    23. ps (see man ps, lists running processes)

    24. kill (see man kill, asks a program to exit, tells you if a program is running, or can force kill a program)

    25. pwd (to view the present working directory)

    26. locate (to locate files)

    27. scp (to copy files to a remote server)

    28. rsync (to sync remote servers)

    29. rpm (for managing packages in Red Hat Enterprise Linux)
